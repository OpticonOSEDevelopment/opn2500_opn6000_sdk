Polling and Timing Options

Description Press on the different areas of the screenshot below to browse through this NetO32 help.
Options
Poll speed
With the poll speed slider you can set the rate at which polls are sent to the handheld terminal(s). The indicator above the slider shows the time the NetO32 applications waits for an answer before transmitting a new poll. The more polls the NetO32 Application transmits per second, the more handheld terminals have an opportunity to start data transfer. But also, the more polls per second are transmitted the more burden the NetO32 application puts on the system. The default value is 0.5 seconds between polls.

Terminals to Poll
Holds the terminals to poll for. If no handheld terminal ID value is entered the broadcast ID is use ("&&&&&&"). When the protocol type is set to Single run type then only the first handheld terminal ID from the list is used. With the Add button a new terminal can be entered. The maximum size for the terminal ID is six characters. A selected handheld terminal ID can be removed by pressing the remove button. The whole list is cleared by clicking the clear button.

Protocol type
The NetO protocol is split up into two types Single and Continuous. The Single run protocol type tries to communicate with a handheld terminal, when communication is stopped by a successful transfer or by an error the protocol needs to be restarted. The Continuous protocol type starts communication and after a successful communication or a not severe error the protocol continues to by polling the next handheld terminal from the list. Default is the continuous protocol type.

Quit after single run
Quit the NetO32 application after a successful data transfer or by an error. This option is only available when the single run protocol type is selected. Default is this option disabled.

Stop after poll timeout
The stop poll timeout options stops the running protocol after a poll timeout event. This options is only available when the single run protocol type is selected. The timeout time is set by the poll timeout time. By default this option is disabled.

Poll timeout (seconds)
Poll timeout time sets the time in seconds after no answer on the poll the protocol must be stopped. This option is only available when the Protocol type is set to Single and the Stop poll timeout check box is checked. The default value for this option is 5 seconds.

Tx/Rx Delay (ms)
The Tx/Rx switch delay specifies the delay time between transmitting and receiving a block of data. This options is needed when using a RS485 multi drop network. The default value for this options is 1 ms.
