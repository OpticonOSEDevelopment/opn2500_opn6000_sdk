Command line options

Description The NetO32 application can be configured from the command line. This way by starting the application some specific behavior of the application can be accomplished. The command line parameters are discussed as they can be found in the options men.
.NSF file When a *.NSF (NetO save File) file is given as command line parameter the previously saved settings of a communication run can be restored.

Example:
Start NetO32 application with a previously saved settings file. 
C:\NetO32>NetO32 testme.nsf
RS232
Option Description
-c: Set the serial COM port to where the terminal is connected. The default serial port number is COM1

Example:
Start NetO32 application with COM2 
C:\NetO32>NetO32 -c:2
-b: Set the communication speed. The default communication speed is 19200 baud.

The communication speed can be one of the following values:

  • 115200 baud
  • 57600 baud
  • 38400 baud
  • 19200 baud
  • 9600 baud
  • 4800 baud
  • 2400 baud

    Example:
    Start NetO32 application with 115200 baud 
    C:\NetO32>NetO32 -b:115200
    • -p: Change the parity setting, default is 0 - No parity.

    The parity option can be one of these values:

  • 0 - No parity
  • 1 - Odd parity
  • 2 - Even parity

    Example:
    Start NetO32 with even parity 
    C:\NetO32>NetO32 -p:2
    • -d: Change the amount of data bits used, default is 8.

    The data bits parameter can be one of these values:

  • 8 data bits
  • 7 data bits

    Example:
    Start NetO32 with 7 data bits 
    C:\NetO32>NetO32 -d:7
    • -s: Set the amount of stop bits default 0 - means one stop bit.

    The stop bits parameter can be one of these values:

  • 0 - one stop bit
  • 2 - two stop bits

    Example:
    Start NetO32 with 2 stop bits 
    C:\NetO32>NetO32 -s:2
    • -i: Set the IrDA adapter or compatible adapter of your choice, default 0 - No IrDA adapter.

    The IrDA adapter parameter can be one of these values:

  • 0 - No IrDA adapter
  • 1 - Litelink compatible
  • 2 - Tekram compatible
  • 3 - ActiSys compatible
  • 4 - Opticon IRU-1300 cradle

    Example:
    Start NetO32 with IrDA adapter set to Opticon IRU-1300 cradle: 
    C:\NetO32>NetO32 -i:4
    • Download
    Option Description
    -dr: Set the folder where the received files are stored. The default folder is the Root of C:

    Example:
    Start NetO32 with the download path set to "c:\opticon" 
    C:\NetO32>Neto32 -dr:c:\opticon
    -dt: Set download receive type. The default value is append the newly received data to the file if it exists.

    The download receive type can be one of these values:

  • 0 - Append
  • 1 - Overwrite
  • 2 - Unique

    Example:
    Start NetO32 in a way that newly received files overwrite existing files 
    C:\NetO32>Neto32 -dt:1
    		•
    Upload
    Option Description
    -u: Set the file(s) that need to be transferred to the terminal. No default value

    Example:
    Start NetO32 with 2 files to transmit to the terminal 
    C:\NetO32>Neto32 -u:c:\opticon\file1.txt -u:c:\opticon\file2.txt
    Polling and Timing
    Option Description
    -ps: Set the poll speed. The default poll speed value is 500.

    The poll speed parameter can be one of these values:

  • 100 - Poll every 0.1 seconds
  • 200 - Poll every 0.2 seconds
  • 500 - Poll every 0.5 seconds
  • 1000 - Poll every second
  • 2000 - Poll every 2 seconds
  • 5000 - Poll every 5 seconds
  • 10000 - Poll every 10 seconds
  • 20000 - Poll every 20 seconds

    Example:
    Start NetO32 with a poll speed of 1 second per poll 
    C:\NetO32>Neto32 -ps:1000
    		•
    -t: Set a specific terminal ID, there is no default value. The terminal ID parameter max size is 6 characters.

    Example:
    Start NetO32 with Terminal ID "123456" 
    C:\NetO32>Neto32 -t:123456
    -rxtx: Set an extra transmit receive delay. This value is needed when using an RS485 multi drop network. The default value is 1ms. Valid input of this parameter lies between 1ms and 999 ms.

    Example:
    Start NetO32 with a Rx/Tx delay of 5 ms 
    C:\NetO32>Neto32 -rxtx:5
    -pt: Set the protocol type, the default value is 0 - Continuous protocol.

    The protocol type parameter can be one of these values:

  • 0 - Continuous
  • 1 - Single

    Example:
    Start NetO32 with the Single run protocol. 
    C:\NetO32>Neto32 -pt:1
    		•
    -qs: Enable or disable the quit after single run. Default is 0 - Do not quit after single run. This command line parameter has only effect when the -pt: command line parameter is set to 1 - Single run.

    The protocol type parameter can be one of these values:

  • 0 - Do not quit after single run
  • 1 - Quit after single run.

    Example:
    Start NetO32 with Quit after single run. 
    C:\NetO32>Neto32 -pt:1 -qs:1
    		•
    -spt: Enable or disable the Poll timeout timer. Default is 0 - Poll timeout timer is not enabled. This command line parameter has only effect when the -pt: command line parameter is set to 1 - Single run.

    The protocol type parameter can be one of these values:

  • 0 - Do not stop after poll timeout
  • 1 - Stop after poll timeout.

    Example:
    Start NetO32 with stop after poll timeout. 
    C:\NetO32>Neto32 -pt:1 -spt:1
    		•
    -spe: Set the Poll timeout time, default value is 5 seconds. This command line parameter has only effect when the -pt: command line parameter is set to 1 - Single run and -spt: command line parameter is set to 1 - stop after poll timeout. The poll timeout time can range between 1 and 32000 seconds.

    Example:
    Start NetO32 with stop poll timeout after 20 seconds. 
    C:\NetO32>Neto32 -pt:1 -spt:1 -spt:20
    Appearance
    Option Description
    -asat: Enable or disable the NetO32 application to start in the system tray. Default 0 - Start not in System Tray.

    The start in system tray can have one of these values:

  • 0 - Start not in system tray
  • 1 - Start in system tray

    Example:
    Start NetO32 in the system tray 
    C:\NetO32>Neto32 -asat:1
    		•
    -asm: Enable or disable the NetO32 protocol message window when the application is minimized to the system tray. The default value is 0 - Show when minimized in system tray.

    The show message window when minimized in system tray can have one of these values:

  • 0 - Do not show message window when minimized
  • 1 - Show message window when minimized

    Example:
    Start NetO32 and show when message window when any protocol status messages are available 
    C:\NetO32>Neto32 -asm:1
    		•
    -asf: Enable or disable the showing of the file progress in the NetO32 protocol message window. The default value is 1 - Show when file progress, this option can only been seen if the -asm: parameter = 1.

    The show file progress message window when minimized in system tray can have one of these values:

  • 0 - Do not file progress
  • 1 - Show file progress

    Example:
    Start NetO32 and show the message window with the file progress bar. 
    C:\NetO32>Neto32 -asm:1 -asf:1
    		•
    -ast: Enable or disable the showing of the total file(s) progress in the NetO32 protocol message window. The default value is 1 - Show when total file(s) progress, this option can only been seen if the -asm: parameter = 1.

    The show total file(s) progress message window when minimized in system tray can have one of these values:

  • 0 - Do not total file(s) progress
  • 1 - Show total file(s) progress

    Example:
    Start NetO32 and show the message window with the total file(s) progress bar. 
    C:\NetO32>Neto32 -asm:1 -ast:1
    		•
    -asi: Enable or disable the showing of the progress status information. The default value is 1 - Show progress status information, this option can only been seen if the -asm: parameter = 1.

    The show progress status information can have one of these values:

  • 0 - Do not progress status information
  • 1 - Show progress status information

    Example:
    Start NetO32 and show a message window with the progress status info. 
    C:\NetO32>Neto32 -asm:1 -asi:1
    		•
    -asp: Start the NetO protocol when the NetO32 application is executed. The default value is 0 - Start NetO protocol at startup.

    The start protocol at startup parameter can have one of these values:

  • 0 - Do not start protocol at startup
  • 1 - Start protocol at startup

    Example:
    Start NetO32 and start the NetO protocol directly. 
    C:\NetO32>Neto32 -asp:1
    		•
    Log
    Option Description
    -lslf: Store messages in a log file. Default 0 - Do not store messages in a log file.

    The store messages in a log file can have one of these values:

  • 0 - Do not store messages in a log file
  • 1 - Store messages in a log file

    Example:
    Start NetO32 and store the messages in a log file 
    C:\NetO32>Neto32 -lslf:1
    		•
    -lfo: Set how to store the options in a log file. Default 0 - Store log in a new log file at startup.

    The log file options can have one of these values:

  • 0 - Store log in a new log file at startup
  • 1 - Store log in a specific log file. With this options the -ls: parameter needs to be set also.

    Example:
    Start NetO32 and store the messages in a new log file at startup 
    C:\NetO32>Neto32 -lslf:1 -lfo:0
    		•
    -ls: Set a specific log file. Default not filled in.

    Example:
    Start NetO32 and store the messages in a specific log file. 
    C:\NetO32>Neto32 -lslf:1 -lfo:1 -ls:c:\log\Neto32.log
    -le: Enable or disable the storing of error messages in a log file. Default 1 - Store Error messages in log file.

    The store error messages in a log file can have one of these values:

  • 0 - Do not store error messages in a log file
  • 1 - Store error messages in a log file

    Example:
    Start NetO32 and store the error messages in a specific log file. 
    C:\NetO32>Neto32 -lslf:1 -lfo:1 -ls:c:\log\Neto32.log -le:1
    		•
    -lc: Enable or disable the storing of completion messages in a log file. Default 1 - Store completion messages in log file.

    The store completion messages in a log file can have one of these values:

  • 0 - Do not store completion messages in a log file
  • 1 - Store completion messages in a log file

    Example:
    Start NetO32 and store the completion messages in a specific log file. 
    C:\NetO32>Neto32 -lslf:1 -lfo:1 -ls:c:\log\Neto32.log -lc:1
    		•
    -lsrd: Enable or disable the storing of send receive messages in a log file. Default 1 - Store send receive messages in a log file.

    The store send receive messages in a log file can have one of these values:

  • 0 - Do not store send / receive messages in a log file
  • 1 - Store send / receive messages in a log file

    Example:
    Start NetO32 and store the send and receive messages in a specific log file. 
    C:\NetO32>Neto32 -lslf:1 -lfo:1 -ls:c:\log\Neto32.log -lsrd:1
    		•
