Log Options

Options
Store Log in file
If enabled, NetO32 will store the log messages into a file. This way later can been seen what problems occurred and when they did happen. By default this option is not enabled.

Log file options
Log file options can set the Log file. The option New log at startup, creates a new log name when the NetO32 application is started. The format of this name is [YYYYMMDDhhmmss].log where Y stand for Year, M for Month,D for Day, h for hour, m for minute and s for second. This way every time the NetO32 application a new file is created and these files are easily sorted for keeping track. The second option is Specific log file. The way the user can select a specific log file to store the log data in. Every time the NetO32 application is started new log data will be appended at the back of this specific file. This option can result in very large files. The default value is New log at startup.

Log errors
If enabled, then NetO32 will store error messages in the Log file and in the status Log of the main NetO32 application window. By default this option is enabled.

Log Completions
If enabled, then NetO32 will store completion messages in the Log file and in the status Log of the main NetO32 application window. By default this option is enabled.

Log send receive data
If enabled, then NetO32 will log the send and receive data messages in the Log file and in the status Log of the main NetO32 application window. By default this options is enabled.
