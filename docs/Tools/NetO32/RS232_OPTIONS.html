RS232 Options

Description Press on the different areas of the screenshot below to browse through this NetO32 help.

Important:

The RS232 settings must all match the settings on the handheld terminal that is being polled by the NetO32 application. When using a communication cradle, also the cradle must be configure on either the same baud rate setting or on auto baud rate.
Options
COM port
Shows all available serial ports that are available on the PC. It also shows serial ports that are in use by other applications. Select in the combo box the serial port to which the handheld terminal(s) is/are connected.

Baud rate
Sets the speed of the selected serial port. The speed selected must match the baud rate that is configured on the terminal. The default value for the speed is "19200" baud, however most application will use a different baud.

Parity
Parity is a simple type of error checking provided by the basic RS232 protocol. The default value for this option is "No parity".

ByteSize
The number of bits in a transmission byte. The default value is "8".

Stop bit(s)
The number of stop bits is a RS232 standard protocol setting. The default value is "1".

IrDA Device
Three standard IrDA devices are supported, and one Opticon specific. If one of these IrDA devices is selected it is possible to connect an IrDA device to the serial port. NetO32 will then configure the connected IrDA device. The default value is "LiteLink compatible", which is the protocol used by most Opticon IrDA cradles to automatically configure the baud rate. This protocl however doesn't effect devices without IrDA.
